Why HydraDB matters

Memory, not decider.

  • ◎
    Similar past cases · posture + focus + app context.
  • ⇄
    Graph context · connected session, app, intervention.
  • ★
    Previous feedback · what this user accepted before.
  • ↗
    Patterns · "walk beat posture-only alerts."
  • ⇢
    Reasoning path · explains the recommendation.
Default

Quiet monitoring

  • Posture stable, focus steady.
  • No consecutive bad captures.
  • State silently logged to HydraDB.
  • UI: "You're in flow."
Trigger

When the agent speaks up

  • Posture worsens past baseline.
  • Focus drops · window switches rise.
  • Consecutive bad captures cross threshold.
  • HydraDB finds an intervention that worked.
